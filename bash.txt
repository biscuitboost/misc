<h1>1. Why Linux and Bash</h1>

<p>Linux and Bash are essential tools for developers:</p>

<ul>
    <li>
        <p>Flexibility:</p>
        <ul>
            <li>Linux provides a highly customizable environment</li>
            <li>Developers can tailor their workspace to their needs</li>
            <li>Bash scripts can automate repetitive tasks, saving time and reducing errors</li>
        </ul>
    </li>
    <li>
        <p>Power:</p>
        <ul>
            <li>Command-line interface offers efficient ways to manipulate and process data</li>
            <li>Often faster than GUI alternatives</li>
            <li>Once you can do something with the command line, you can automate it</li>
            <li>Automation with GUI elements like sliders is much more challenging</li>
        </ul>
    </li>
    <li>
        <p>Ubiquity in development:</p>
        <ul>
            <li>Linux is widely used in servers, cloud environments, and DevOps practices</li>
            <li>Familiarity with Linux is crucial for deploying and managing Java applications in production environments</li>
        </ul>
    </li>
</ul>

<h1>2. Unix Architecture (simple understanding)</h1>

<p>Basic components of Unix-like systems (including Linux):</p>

<ul>
    <li>
        <p>Kernel:</p>
        <ul>
            <li>Core of the operating system</li>
            <li>Manages hardware resources</li>
            <li>Schedules processes</li>
            <li>Provides essential services</li>
        </ul>
    </li>
    <li>
        <p>Shell:</p>
        <ul>
            <li>Interface between the user and the kernel</li>
            <li>Bash (Bourne Again Shell) is one of the most popular shells</li>
            <li>Many new shells exist, but BASH is good to understand due to its wide use</li>
        </ul>
    </li>
    <li>
        <p>File System:</p>
        <ul>
            <li>"Everything is a file" philosophy</li>
            <li>Organized in a hierarchical structure</li>
            <li>Root directory (/) at the top</li>
            <li>Examples of "files":
                <ul>
                    <li>Regular files</li>
                    <li>Directories</li>
                    <li>Device files (e.g., /dev/null)</li>
                    <li>Hardware serial devices</li>
                    <li>NFS file mounts</li>
                </ul>
            </li>
        </ul>
    </li>
</ul>
